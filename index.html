First Contentful Paint: 5s

The time it takes before content begins appearing on the page.

5 seconds is a bit misleading as these testing tools simulate a slow connection speed or "slow" device (in this case a MotoG4). When testing by eye on my MacBook and iPhone the page renders within a second. But these tools are useful in providing a metric that can be tested and improved.

Render Blocking Resources

Elements that prevent a web page from being fully rendered by the browser until they are completely loaded and processed.

I would look into these files to see if it's possible to load them later in the load cycle. Unfortunately many of these files come from 3rd party apps which we do not have access to (see below: gorgias chat, afterpay, yotpo), and Shopify does not give us access to the order in which our Shopify theme files are loaded.

Summary

  1. Apps: Using many apps is impacting load speed (this is normal for a Shopify store).

    According to Google Page Speed Insights and Web Page Test, 3rd party apps are a big part of decreased load speed. Unfortunately there is not much we can do about this since we do not have access to the external, 3rd party app code. Recommendations below.

  2. Render Blocking Resources

    The files in the image below (before the green line) are blocking the page from rendering until they are loaded. While this is normal, and many of these files need to be loaded first to prevent things like showing unstyled content, I would see if there are any opportunities here to load some of these files at the end of page load rather than at the beginning.

Recommendations

  1. Apps: Removing Non-Vital Apps / Offloading Any Unused Apps

    I would take an inventory of any apps not required or not being used on the site and delete them from the store.

    Using a lot of 3rd party apps on a Shopify site will considerably slow down load speed. But nfortunately many apps are vital to store functionality, so this is out of our control (since these are 3rd party and we do not have access to their code).

  2. Lazy Loading

    Lazy Loading is the process of loading images when they come into view in the viewport, rather than loading all images on a page at once.

    I'm seeing that all images on the page, even though they are tagged with the loading="lazy" or loading="eager" attributes, seem to be loading all at once and don't seem to be lazy loading.

    This may be due to the use of Content Stack to load the images, so it may not be possible to lazy load the images (since they are coming from a 3rd party), but I would look into adding a lazy loading script into the shopify store to see if the images can be lazy loaded.

  3. Removing Render Blocking Resources

    The way I would begin trying to improve load speed is to start by trying to improve the First Contentful Paint, which is the time it takes for text or images to appear on the screen, which Google Page Speed and Web Page Test clock at 5 seconds for mobile.

    The way I would do this is by going through the list of recommendations offered by Page Speed Insights and Web Page Test and see which files are available to be optimized (any non-3rd party files).
